# CRM Frontend Agents

## Purpose

- define how PM, CTO, DEV, and QA work inside the frontend surface
- keep UI work aligned with the CRM design system and quality bar

## Frontend Role Focus

### PM

- shapes frontend slices and acceptance criteria
- keeps UI docs and sprint tasks aligned

### CTO

- defines component boundaries and state strategy
- reviews rendering, data flow, and performance tradeoffs

### DEV

- implements components, routes, styles, and frontend tests
- follows existing UI patterns and accessibility requirements

### QA

- verifies user-visible behavior, regressions, and responsive behavior
- checks accessibility and critical frontend flows

## Frontend Rules

- components render and compose UI
- hooks and utilities own reusable logic
- preserve accessibility and keyboard support
- test user-visible flows, not only helpers
- follow the design system and UI kit
